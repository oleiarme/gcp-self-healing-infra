# План исправления: Отсутствующий секрет cf_token

## Проблема
При запуске `scripts/startup_cos.sh` возникает ошибка:
```
❌ Missing secret: cf_token (secret fetch may have partially failed)
```

## Причина
В скрипте `startup_cos.sh` секреты `db_password` и `n8n_key` записываются в `/dev/shm/n8n-secrets/`, но `cf_token` — нет. Проверка на строке 319 не находит файл и прерывает выполнение.

## Решение
Внести изменения в файл `scripts/startup_cos.sh`:

1. **Место для правки:** После строки 316 (`umask 022`), перед проверкой секретов.
2. **Действие:** Добавить запись токена Cloudflare в файл.

```bash
# Текущий код (строки 312-316):
umask 077
mkdir -p /dev/shm/n8n-secrets
printf "%s" "$DB_PASSWORD" > /dev/shm/n8n-secrets/db_password
printf "%s" "$N8N_KEY" > /dev/shm/n8n-secrets/n8n_key
umask 022

# Нужно добавить после umask 022:
printf "%s" "$CF_TOKEN" > /dev/shm/n8n-secrets/cf_token
```

## Ожидаемый результат
Скрипт успешно создаст файл с токеном Cloudflare, проверка секретов пройдет успешно, и контейнер `cloudflared` запустится с корректным токеном.
