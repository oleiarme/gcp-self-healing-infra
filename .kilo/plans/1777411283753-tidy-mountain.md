# Fix MQL Unit Mismatch in uptime_latency_high Alert Policy

## Problem
The Terraform deployment fails with an MQL warning about unit mismatch in `monitoring.tf:476`:
- `p95` value has units of 'ms' (from `request_latency` metric)
- Literal `2000` has no units
- MQL requires both sides of `>` to have units or neither

## Solution
Replace `cast_units(2000, "ms")` with proper MQL literal unit annotation: `2000 "ms"`

## Changes
**File:** `terraform/monitoring.tf` (line 476)

**Current:**
```mql
| condition p95 > cast_units(2000, "ms")
```

**Fixed:**
```mql
| condition p95 > 2000 "ms"
```

## Verification
- Run `terraform validate` to check syntax
- Run `terraform plan` to verify the alert policy creates successfully
